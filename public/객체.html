객체에 대해 알아보자

객체의 기본

  • 객체는 여러 개의 자료형을 한 번에 저장하는 자료형이다.
  • 여러 개를 저장할 수 있는 배열도 객체를 기반으로 한다.
  • 배열은 요소에 접근 시 인덱스를 사용하고, 객체는 키를 사용한다.
    • 배열의 인덱스는 숫자, 객체의 키는 식별자, 문자열, 또는 심볼
    • 예) 배열 a의 요소에 접근 : a[3]
    • 예) 객체 b의 요소에 접근 : b["name"] 또는 b.name (* 식별자로 사용할 수 없는 키일 경우는 대괄호 방식만 지원된다.)
  • 객체 생성

요소와 속성

  • 자바 스크립트에서는 요소와 속성의 구분이 큰 의미가 없다.
  • 일반적으로 요소는 배열의 각 아이템을 의미하고, 속성은 객체의 각 아이템을 의미한다.
  • 자바스크립트는 배열과 객체의 요소에 다양한 자료형을 입력할 수 있다.
  • 자료형이 함수인 속성을 메소드라고 한다.
  • 메소드에서 다른 속성을 접근할 때 this를 사용한다.

생성자 함수와 프로토타입

  • 일반적인 프로그래밍은 자료를 다루는 것이 목적이고, 그래서 현실의 객체를 모방해서 프로그래밍하는 '객체 지향 프로그래밍'의 패러다임을 갖는 언어가 주류를 이룬다.
  • 우리가 익숙한 일반적인 OOP는 클래스 기반 객체지향
    • 객체의 자료 구조와 기능을 클래스로 정의하고, 생성자를 통해 인스턴스를 생성
  • 자바스크립트는 프로토타입기반 객체지향
    • 생성자 함수를 통해 객체를 생성
    • dynamic 언어답게 이미 생성된 인스턴스의 자료 구조와 기능을 변경할 수 있음
    • 상송, 캡슐화 등의 객체지향 개념도 프로토타입 체인이나 클로저 등으로 구현 가능
  • 프로토타입
    • 자신의 부모 역할을 담당하는 객체로서 모든 객체에 연결되어있음

생성자 함수 개요

  • 배열과 객체를 사용하면 여러 개의 데이터를 쉽게 다룰 수 있다.
  • 앞의 객체에 print 메소드를 추가한다.
  • 함수를 따로 빼면 간단하지만 객체지향이 아니다.
  • 객체를 만드는 함수로서 일반 함수와 구분을 위해 보통 이름 첫 글자는 대문자로 쓴다.

프로토타입

자신의 부모 역할을 하는 객체로서 생성자 함수로 만든 모든 객체에 연결되어있음

클래스(ES6)

프로토타입 기반의 객체 지향은 자유도가 높아서 오히려 개발하기 어려운 면이 있었기에 ES6부터 class 문법이 추가됨

Null

  • 원시 자료형 중 하나일 것 같지만, null 값을 가지는 객체이다.
  • typeof(null)=='object'
  • 비교 연산자 사용 시 주의가 필요하다.
    • if(a==null)
    • if(a==undefined)
    • 위 셋은 모두 같은 결과를 준다.
    • a가 null 이거나 undefined일 경우만 true로 판정된다.

Symbol(ES6)

  • ES6에서 추가된 새로운 primitive data type
  • Symbol() 함수는 symbol type의 값을 반환
  • new Symbol() 문법은 지원하지 않음
  • Symbol()로부터 반환되는 모든 값은 고유함
  • 심볼 값은 객체의 속성키의 식별자로 사용가능
    • 사실 이것이 유일한 존재 이유
